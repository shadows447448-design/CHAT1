# PRD：合法远程访问场景的一键部署 WireGuard VPN 工具

## 1. 文档信息
- **产品名称**：WireGuard One-Click Deploy（暂定）
- **版本**：v1.0（MVP）
- **文档类型**：产品需求文档（PRD）
- **目标**：为“合法远程访问”场景提供可审计、可回滚、低门槛的一键化 WireGuard VPN 部署与运维工具。

---

## 2. 背景与问题定义

### 2.1 背景
中小团队、个人开发者、远程运维人员常需要在合法授权的前提下，安全访问内网资源（如开发环境、测试环境、家庭实验室、受控服务器）。

### 2.2 现有痛点
1. WireGuard 手工部署步骤多（安装、转发、NAT、防火墙、密钥管理）。
2. 客户端配置分发和吊销流程不统一，容易遗漏。
3. 出问题后排障成本高（难以快速确认接口状态、路由、防火墙规则）。
4. 不规范日志与配置管理导致安全风险（明文密钥、敏感信息泄露）。

### 2.3 目标结果
通过 CLI 一键工具将部署和基础运维流程标准化，使用户可以在 10 分钟内完成安全、可用、可回滚的 WireGuard 服务端上线。

---

## 3. 目标用户与核心场景

### 3.1 目标用户
1. **个人开发者 / 技术爱好者**：需要从外网安全访问家庭或个人云主机。
2. **小团队运维/后端工程师**：需要快速为开发/测试环境搭建受控远程访问。
3. **技术支持人员**：在获得授权的前提下，临时接入客户/分支环境进行排障。

### 3.2 核心场景
1. **首次部署场景**：在一台 Linux 服务器上执行单条命令完成 WireGuard 服务端安装和初始化。
2. **新增客户端场景**：快速生成客户端配置（含二维码/配置文件），并设置访问范围。
3. **状态巡检场景**：一键查看服务是否在线、端口是否监听、Peer 是否握手成功。
4. **安全回收场景**：员工离职或设备丢失后，快速吊销客户端访问。
5. **迁移与恢复场景**：升级或迁移前备份配置，故障后可恢复。

---

## 4. 产品目标与非目标

### 4.1 产品目标（MVP）
1. 提供统一 CLI 命令，覆盖安装、配置生成、客户端管理、状态检查、卸载。
2. 默认提供安全基线（最小权限、密钥文件权限控制、敏感日志脱敏）。
3. 支持基本备份与恢复流程，降低误操作风险。
4. 输出清晰的执行结果与错误提示，便于自助排障。

### 4.2 非目标（Out of Scope）
1. 不提供“绕过监管/审查/地理限制”等用途能力。
2. 不支持多区域高可用集群、自动弹性扩缩容（MVP 阶段）。
3. 不包含完整 Web 控制台（MVP 仅 CLI + 配置文件）。
4. 不负责第三方终端 App 的安装与兼容问题（仅生成标准 WireGuard 配置）。

---

## 5. MVP 范围

### 5.1 平台与环境
- **服务端系统**：Ubuntu 22.04+/Debian 12+（MVP 首发）
- **运行方式**：Shell 脚本或单文件二进制 CLI
- **依赖**：`wireguard-tools`、`iproute2`、`iptables`/`nft`

### 5.2 交付能力
- 一键初始化服务端（含 IP 转发和防火墙规则）
- 客户端增删改查（重点：新增、吊销）
- 状态检查与健康摘要
- 卸载与回滚（尽量清理残留）
- 配置备份与恢复（至少备份服务端配置与客户端索引）

---

## 6. 功能需求（按模块）

## 6.1 模块 A：安装（Install）

### 功能描述
自动完成 WireGuard 服务端部署与基础网络配置。

### 关键能力
1. 依赖检查：检测系统版本、内核模块、工具依赖。
2. 自动安装：安装 WireGuard 及必需组件。
3. 初始化网络：
   - 生成服务端地址段（如 `10.8.0.1/24`）
   - 开启 IPv4 转发
   - 配置 NAT 与转发规则
4. 端口与接口初始化：
   - 默认端口（如 51820，可自定义）
   - 默认接口名（如 `wg0`，可自定义）
5. 首次启动与自启配置：`systemd` 启动、开机自启。

### 输入
- 监听端口
- 公网网卡名（可自动识别并允许确认）
- VPN 网段

### 输出
- 安装成功/失败状态
- 服务端公钥指纹
- 下一步操作指引（新增客户端命令）

---

## 6.2 模块 B：配置生成（Config Generation）

### 功能描述
生成服务端与客户端配置，并保证配置可读、可追踪、可恢复。

### 关键能力
1. 自动生成密钥对（服务端/客户端）。
2. 生成标准 WireGuard 配置文件：
   - 服务端：`wg0.conf`
   - 客户端：`client-<name>.conf`
3. 地址分配策略：自动分配未占用地址；支持手动指定。
4. 配置模板化：DNS、AllowedIPs、PersistentKeepalive 可配置。
5. 导出能力：
   - 文本配置导出
   - 二维码输出（终端 ASCII 或图片）

### 输入
- 客户端名称（唯一）
- 客户端地址（可选）
- 客户端路由策略（如仅内网或全流量）

### 输出
- 客户端配置文件路径
- 二维码/复制指令
- 客户端索引记录（名称、公钥、分配 IP、创建时间）

---

## 6.3 模块 C：客户端管理（Client Management）

### 功能描述
对接入客户端进行生命周期管理，确保可控可审计。

### 关键能力
1. 新增客户端：创建密钥、分配地址、写入服务端 peer。
2. 列表查看：展示客户端状态（已配置/最近握手时间/流量统计）。
3. 吊销客户端：从服务端配置移除 peer，并保留审计记录。
4. 重置客户端：支持重新生成密钥并替换配置。
5. 命名规范校验：防止特殊字符导致配置注入风险。

### 输入
- 客户端名称
- 操作类型（add/list/revoke/reset）

### 输出
- 操作结果
- 变更摘要（受影响 peer 数量、配置版本）

---

## 6.4 模块 D：状态检查（Status Check）

### 功能描述
提供一键健康检查，减少排障复杂度。

### 关键能力
1. 服务状态：`wg-quick@wg0` 是否运行。
2. 网络状态：端口监听、防火墙规则、IP 转发开关。
3. Peer 状态：最近握手时间、收发流量、异常 peer 提示。
4. 诊断建议：按问题类型给出修复建议（端口未放行、网卡错误等）。

### 输入
- 检查范围（quick/full）

### 输出
- 结构化健康报告（通过/警告/失败）
- 可执行建议命令

---

## 6.5 模块 E：卸载（Uninstall）

### 功能描述
安全移除工具及相关配置，防止遗留风险。

### 关键能力
1. 卸载前确认：二次确认并提示影响范围。
2. 自动备份：卸载前可选自动备份。
3. 停止服务：关闭 WireGuard 服务。
4. 清理配置：删除工具生成配置与规则（可保留备份）。
5. 结果报告：输出残留项清单（如用户手工改动规则）。

### 输入
- 卸载模式（保留备份/彻底清理）

### 输出
- 卸载结果
- 备份位置
- 残留提示

---

## 7. 安全要求

### 7.1 最小权限（Least Privilege）
1. CLI 默认以普通用户执行，仅在必要步骤通过 `sudo` 提升权限。
2. 将高权限操作限定在最小命令集合（安装、写系统配置、启动服务）。
3. 工具内部按步骤拆分权限，避免全流程 root 常驻执行。

### 7.2 密钥保护（Key Protection）
1. 私钥禁止打印到终端标准输出。
2. 私钥文件权限必须为 `600`，目录权限最小化（建议 `700`）。
3. 禁止将私钥写入通用日志。
4. 支持密钥轮换（至少支持客户端重置）。

### 7.3 日志脱敏（Log Redaction）
1. 日志中屏蔽私钥、完整配置内容、完整公网 IP+密钥组合信息。
2. 错误日志默认仅输出必要上下文，详细调试日志需显式开启。
3. 调试模式下也必须对私钥字段做不可逆掩码。

### 7.4 配置备份（Config Backup）
1. 提供手动/自动备份能力（安装后、变更前、卸载前）。
2. 备份文件包含元信息（时间戳、版本、校验和）。
3. 恢复前先进行格式与完整性校验。
4. 支持“仅恢复客户端索引”与“完整恢复”。

---

## 8. 非功能需求

### 8.1 可用性
- 首次部署流程交互步骤不超过 8 步。
- 关键命令提供 `--help` 示例。

### 8.2 可靠性
- 配置写入采用“临时文件 + 原子替换”策略，避免半写入损坏。
- 关键变更前自动创建回滚点。

### 8.3 可维护性
- 配置文件与状态索引分离。
- 所有核心操作输出统一错误码。

### 8.4 可审计性
- 对客户端新增/吊销操作记录最小审计信息（时间、操作者、目标客户端）。

## 9. 命令行交互草案

```bash
wg-ocd install
wg-ocd client add alice
wg-ocd client list
wg-ocd client revoke alice
wg-ocd status --full
wg-ocd backup create
wg-ocd restore --latest
wg-ocd uninstall --keep-backup
```

## 10. 验收标准（Acceptance Criteria）

### 10.1 安装
1. 在支持系统上执行 `install` 后，服务 2 分钟内可用。
2. `wg show` 可看到服务端接口与监听端口。
3. IP 转发与 NAT 规则正确生效。

### 10.2 配置生成
1. 新增客户端后自动生成可导入配置文件。
2. 客户端名称重复时被拒绝并提示原因。
3. 地址冲突时自动避让或明确报错。

### 10.3 客户端管理
1. `client list` 能显示全部已注册客户端及最近握手时间。
2. 吊销客户端后，该客户端 1 分钟内无法继续建立握手。
3. 客户端重置后旧配置失效，新配置可连接。

### 10.4 状态检查
1. `status --full` 至少输出：服务状态、端口状态、peer 状态、问题建议。
2. 当端口未放行时，检查报告能明确指出并给修复命令示例。

### 10.5 卸载
1. 卸载后 `wg-quick@wg0` 服务停止并禁用。
2. 选择保留备份时，备份可被 `restore` 正常识别。
3. 选择彻底清理时，工具生成的配置文件与规则被移除。

### 10.6 安全
1. 任意命令执行过程中不应在日志或终端输出私钥明文。
2. 密钥文件权限检查通过（私钥 `600`）。
3. 调试日志中的敏感字段均被掩码。
4. 每次关键变更（新增/吊销/卸载）前自动触发备份或提示确认。

---

## 11. 里程碑建议
1. **M1（第 1 周）**：安装 + 状态检查 MVP。
2. **M2（第 2 周）**：配置生成 + 客户端新增/列表。
3. **M3（第 3 周）**：客户端吊销/重置 + 备份恢复。
4. **M4（第 4 周）**：卸载流程 + 安全审计 + 文档完善。

---

## 12. 风险与缓解
1. **系统差异风险**（iptables/nft 差异）：增加环境探测与分支策略。
2. **误操作风险**（误删配置）：所有破坏性操作默认二次确认与自动备份。
3. **安全风险**（密钥泄露）：统一敏感字段处理组件 + 权限巡检命令。
4. **网络不可达风险**（云安全组未放行）：状态检查内置云侧检查提示模板。

---

## 13. 发布标准（Go/No-Go）
满足以下条件方可发布 MVP：
1. 验收标准通过率 ≥ 95%。
2. 高危安全缺陷（密钥泄露、权限失控）为 0。
3. 文档完整（安装、升级、回滚、故障排查）。
4. 在至少两种目标发行版完成端到端验证。
